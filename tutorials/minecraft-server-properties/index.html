    服务器配置文件解析

    适用于Minecraft 1.18 服务器

    server.properties

    1. enable-jmx-monitoring

      类型：布尔值

      默认值：false

      暴露一个具有对象名net.minecraft.server:type=Server的MBean和两个属性averageTickTime和tickTimes用于暴露以毫秒为单位的tick时间

      如果你要启用JMX, 请前往Monitoring and Management Using JMX Technology查阅详细参数设置

    2. rcon.port

      类型：整型（1-65534）

      默认值：25575

      设置RCON远程访问的端口号（前提是enable-rcon的值为true）

    3. gamemode

      类型：字符串/整型（0-3）

      默认值：survival

      定义默认游戏模式

      survival (0) - 生存模式

      creative (1) - 创造模式

      adventure (2) - 冒险模式

      spectator (3) - 旁观模式

    4. enable-command-block

      类型：布尔值

      默认值：false

      是否启用命令方块

    5. enable-query

      类型：布尔值

      默认值：false

      是否允许使用GameSpy4协议的服务器监听器

    6. level-name

      类型：字符串

      默认值：world

      此参数的值将作为世界名称及其文件夹名（存档）

      Warning

      '（单引号）需要在前面加反斜杠号\才能被检测到

    7. motd

      类型：字符串

      默认值：A Minecraft Server

      玩家客户端的多人游戏服务器列表中显示的服务器信息，显示于名称下方

      支持样式代码、特殊符号，但不能超过59个字符

    8. query.port

      类型：整型（1-65534）

      默认值：25565

      设置监听服务器的端口号（前提是enable-query的值为true）

    9. pvp

      类型：布尔型

      默认值：true

      是否允许玩家互相残杀

      Warning

      间接伤害无法因此避免

    10. difficulty

      类型：字符串/整型(0-3)

      默认值：easy

      世界难度：

      peaceful (0) - 和平

      easy (1) - 简单

      normal (2) - 普通

      hard (3) - 困难

    11. network-compression-threshold

      类型：整数（尽量在64-1500之间）

      默认值：256

      wiki的太复杂了，说白了就是：

      如果你网好，值设大一点；否则就小点

      原理：

      默认会允许n-1字节的数据包正常发送, 如果数据包为n字节或更大时会进行压缩。所以，更低的数值会使得更多的数据包被压缩，但是如果被压缩的数据包字节太小将反而使压缩后字节更大

    12. require-resource-pack

      类型：字符串(URL)

      默认值：false

      指向一个资源包的URL，玩家可选择是否使用该资源包

    13. max-tick-time

      类型：整型（0 - 2^63-1）

      默认值：60000

      每个tick花费的最大毫秒数

      若服务器的每个tick超过所设置的毫秒数，服务器将会自动崩溃

      你可以设置为-1，禁用此参数

    14. use-native-transport

      类型：布尔值

      默认值：true

      是否使用针对Linux平台的数据包收发优化

    15. max-players

      类型：整型（0-2147483647）

      默认值：20

      服务器同时能容纳的最大玩家数量

      Warning

      此数值越大，服务器负担越重

    16. online-mode

      类型：布尔值

      默认值：true

      是否让服务器对比Minecraft账户数据库验证登录信息

      Warning

      如果你启用了yggdrasil类似的第三方登录，参数为 true

    17. enable-status

      类型：布尔值

      默认值：true

      是否使服务器在服务器列表中看起来是在线的

    18. allow-flight

      类型：布尔值

      默认值：false

      如果是 false，则生存模式玩家悬空超过5秒就被踢出，如果是 true，就不会

    19. broadcast-rcon-to-ops

      类型：布尔值

      默认值：true

      是否向所有在线OP发送通过RCON执行的命令的输出

    20. view-distance

      类型：整型（3-32）

      默认值：10

      设置服务端发送给客户端的世界数据量，也就是设置玩家各个方向上的区块数量（以玩家为中心的半径）

    21. server-ip

      类型：字符串(IP)

      默认值：无

      将服务器与一个特定IP绑定，建议留空

    22. resource-pack-prompt

      类型：字符串

      默认值：无

      在使用require-resource-pack时在资源包提示界面显示自定义信息

      与聊天组件语法一致，可以包含多行文本

    23. allow-nether

      类型：布尔值

      默认值：true

      是否允许玩家通过下界传送门前往下界

    24. server-port

      类型：整型（1-65534）

      默认值：25565

      服务器（监听的）端口号

    25. enable-rcon

      类型：布尔值

      默认值：false

      是否允许远程访问服务器控制台

      Warning

      RCON协议未加密，谨慎使用

    26. sync-chunk-writes

      类型：布尔值

      默认值：true

      是否允许区块文件以同步模式写入

    27. op-permission-level

      类型：整型（1-4）

      默认值：4

      OP的权限等级

    28. prevent-proxy-connections

      类型：布尔值

      默认值：false

      如果服务器发送的ISP/AS和Mojang的验证服务器的不一样，玩家将会被踢出

      说白了就是是否禁止玩家使用代理或者VPN进入服务器

    29. hide-online-players

      类型：布尔值

      默认值：false

      是否让服务端在响应客户端状态请求时不会返回在线玩家列表

    30. resource-pack

      默认值：无

    31. entity-broadcast-range-percentage

      类型：整型（10-1000）

      默认值：100

      实体需要距离玩家有多近才会将数据包发送给客户端

      数值越高，实体可以在更远的地方被渲染，同时会增加延迟

    32. simulation-distance

      类型：整型（3-32）

      默认值：10

      设置服务端可更新实体范围的最大值，即玩家各个方向上的区块数量（以玩家为中心的半径）

    33. rcon.password

      类型：字符串

      默认值：无

      设置RCON远程访问的密码(前提是enable-rcon的值为true)

    34. player-idle-timeout

      类型：整型

      默认值：0

      如果不为0，服务器将在玩家的空闲时间达到设置的时间（单位：分钟）时将玩家踢出服务器

      注：当服务器接受到下列数据包之一时将会重置空闲时间：点击窗口、附魔物品、更新告示牌、玩家挖掘方块、玩家放置方块、更换拿着的物品、动画(挥动手臂)、实体动作、客户端状态、聊天信息、使用实体

    35. debug

      默认值：false

      是否开启调试模式

    36. force-gamemode

      类型：布尔值

      默认值：false

      是否强制玩家加入时为默认游戏模式

    37. rate-limit

      类型：整型

      默认值：0

      设置玩家被踢出服务器前，可以发送的数据包数量

      设置为0表示关闭此功能

    38. hardcore

      类型：布尔值

      默认值：false

      如果设为 true，服务器难度的设置会被忽略并且设为困难，玩家在死后会自动切换至旁观模式

    39. white-list

      类型：布尔值

      默认值：false

      是否启用服务器的白名单

      Warning

      OP自动视为白名单玩家

    40. broadcast-console-ops

      类型：布尔值

      默认值：true

      是否向所有在线OP发送所执行命令的输出

    41. spawn-npcs

      类型：布尔值

      默认值：true

      是否允许生成村民

    42. spawn-animals

      类型：布尔值

      默认值：true

      是否允许自然生成动物

    43. function-permission-level

      类型：整型（1-4）

      默认值：2

      设定函数的默认权限等级

    44. text-filtering-config

      类型：Json（猜测）

      官方Wiki没有给出资料，我在一个人的博客上找到了相关研究： Walshy的博客

      默认值：无

      这个参数是设置屏蔽词的

      值的格式是{"apiServer":"http(s)://地址:端口号","apiKey":"XXXXXXXX","ruleId":?,"serverId":"test","hashesToDrop":?,"maxConcurrentRequests":?}

    45. spawn-monsters

      类型：布尔值

      默认值：true

      是否允许攻击型生物生成

    46. enforce-whitelist

      类型：布尔值

      默认值：false

      是否在服务器上强制执行白名单

      true - 不在白名单上的用户会被踢出

      false - 不在白名单上的在线用户不会被踢出

    47. resource-pack-sha1

      类型：字符串

      默认值：无

      资源包的SHA-1值，必须为小写十六进制；尽管不能验证资源包的完整性，但是它提高了资源包缓存的有效性和可靠性

    48. spawn-protection

      类型：整型

      默认值：16

      通过将该值进行2x+1的运算来决定出生点的保护半径（如果没有OP，此参数无效）

      设置为0禁用此参数

    49. max-world-size

      类型：整型（1-29999984）

      默认值：29999984

      可让世界边界获得的最大半径值（单位：方块）

      若设置的参数超过默认值，将不会起任何效果

    bukkit.yml

    settings

    一般 Bukkit服务器选项

    1. allow-end

      类型：布尔值

      默认值：true

      是否启用末地

    2. warn-on-overload

      类型：布尔值

      默认值：true

      是否在控制台显示[WARNING] Can't keep up! Did the system time change, or is the server overloaded?此类警告信息

    3. permissions-file

      类型：字符串（文件名）

      默认值：permissions.yml

      设置权限文件

    4. update-folder

      类型：字符串

      默认值：update

      放置更新的插件的文件夹名称，它将在重启时被移动

      Warning

      该文件夹必须在插件文件夹(plugins)中，且必须为相对路径

    5. plugin-profiling

      类型：布尔值

      默认值：false

      是否允许使用/timings命令

      用来测量插件为事件所花费的时间

    6. connection-throttle

      类型：整型

      默认值：4000

      客户端在最近一次连接尝试后被允许再次连接的延迟

      不建议值为0，容易受到攻击

    7. query-plugins

      类型：布尔值

      默认值：true

      服务器在远程查询时是否返回插件列表

    8. deprecated-verbose

      类型：字符串

      默认值：default

      当一个插件注册一个废弃的事件时，服务器是否显示警告

    9. shutdown-message

      类型：字符串

      默认值：Server closed

      服务器停止时显示给客户端的信息

    10. minimum-api

      无资料

      类型：字符串

      默认值：none

    spawn-limits

    Bukkit服务器选项，决定每个世界可以产生多少动物或怪物

    Warning

    类型均为整型

    1. monsters

      默认值：70

      允许服务器设置每个世界可以产生的怪物数量

    2. animals

      默认值：10

      允许服务器设置每个世界可以产生的动物数量

    3. water-animals

      默认值：5

      允许服务器设置每个世界可以产生的水生动物数量

    4. water-ambient

      默认值：20

      允许服务器设置每个世界可以产生的环境生物（又称蝙蝠）的数量

    5. water-underground-creature

      无资料

      默认值：5

    Warning

    这些限制只适用于加载块中的动物或怪兽，它不考虑储存在未加载块中的实体

    chunk-gc

    将检查那些本应被卸载但由于某种原因而未能卸载的区块

    Warning

    类型均为整型

    1. period-in-ticks

      默认值：600

      区块垃圾回收考虑的间隔时间

      如果设置为0，区块垃圾回收将被禁用

    ticks-per

    决定了特定功能的勾选延迟

    Warning

    类型均为整型

    1. animal-spawns

      默认值：400

      允许服务器设置怪物出生时的延迟时间(tick)

    2. monster-spawns

      默认值：1

      允许服务器设置动物出生时的延迟时间(tick)

    3. water-spawns

      默认值：1

      允许服务器设置水生动物出生时的延迟时间(tick)

    4. water-ambient-spawns

      默认值：1

    5. water-underground-creature-spawns

      默认值：1

    6. ambient-spawns

      默认值：1

      允许服务器设置环境生物（类似蝙蝠）出生时的延迟时间(tick)

    7. autosave

      默认值：6000

      允许服务器设置每个save-all事件的ticks数量

    aliases

    Warning

    在最新的版本中，此功能已经被移到commands.yml文件中

    官方示例文件： 

    aliases:
    # This creates a new command called "canihasbukkit", which when used it actually performs "version"
    canihasbukkit: version

    # And this will execute both "save-all" and then "stop" when you type "savestop"
    savestop: [save-all, stop]

    # If a command already exists called "give", it will be rewritten. This is handy if you've two plugins
    # fighting for the same name.
    give: somepluginsgive
